Meeting the brief

Basic Requirements:

1. Environmental Monitoring System : A micro: bit is programmed to use environmental sensors (humidity, temperature, light) to monitor conditions contributing to wildfire risk, calculate a risk score according to the readings and buzz based on this score.

2. Data Validation and Storage : Sensor data is recorded periodically every 2 seconds via a serial connection.

3. Heat Rising Simulation : I heat the system up using steam from boiling water. The system displays a risk score on an LED screen and triggers buzzer alerts of different tones when risk reaches thresholds of 3 (middle) or 5 (high).

Advanced Requirements:

1. Predictive Python Model : A Python model was created based on a scoring system designed to analyse wildfire risk considering factors like temperature, humidity, and simulated data.

2. "What If" Scenario Analysis : The model is capable of answering "what if" scenarios, such as simulating severe drought conditions or drastic temperature rises, to analyse changes in risk levels.

3. Data Comparison and Logging : The system reads a CSV file containing time-stamped input data to calculate risks and compare them against previous results to validate model accuracy.
